# Support

Toolkitty is maintained as a pre-release alpha project. Community support is best effort; there is no guaranteed response time, compatibility promise, or production support agreement.

## Where to ask

- Reproducible application or build bugs: use the [bug report form](https://github.com/0cwa/toolkitty/issues/new?template=bug.yml).
- Feature proposals: use the [feature request form](https://github.com/0cwa/toolkitty/issues/new?template=feature.yml).
- Security-sensitive reports: follow [SECURITY.md](SECURITY.md), not the public issue tracker.

Before opening an issue, search existing issues and review the setup in [README.md](README.md) and [CONTRIBUTING.md](CONTRIBUTING.md).

## Helpful diagnostic information

Include the Toolkitty commit or version, operating system and architecture, Node/npm/Rust versions, exact command, expected result, actual result, and minimal reproduction steps. Paste short logs as text with secrets and personal data removed. For UI issues, screenshots are useful when they do not expose private calendar or identity data.

General Tauri, Svelte, Rust, operating-system packaging, or p2panda questions may belong in the relevant upstream project if they can be reproduced independently of Toolkitty.
